# MediVoice FastAPI Backend

This backend handles voice triage orchestration for MediVoice.

## What it does
- Accepts real-time transcripts from the web app
- Uses the Vapi web SDK for browser voice capture
- Uses Qdrant for symptom / case retrieval
- Returns a clinical triage summary with likely problem, severity, and guidance

## Qdrant Retrieval Mode
- Semantic mode (default): uses FastEmbed (`BAAI/bge-small-en-v1.5`) to generate vector embeddings.
- Fallback mode: if semantic embedding is unavailable, backend falls back to deterministic hash embeddings to avoid downtime.

Environment flags in `backend/.env`:
- `QDRANT_SEMANTIC_ENABLED=true|false`
- `QDRANT_EMBEDDING_PROVIDER=fastembed`
- `QDRANT_EMBEDDING_MODEL=BAAI/bge-small-en-v1.5`

Important:
- `QDRANT_VECTOR_SIZE` must match your collection vector size.
- If you switch embedding model or vector size, re-index your Qdrant collection for best relevance.

## Drug Interaction Checker
- The dedicated drug checker uses a separate Qdrant collection: `medivoice_drugs`.
- It exposes `POST /drug-checker/analyze-text` for voice-transcript drug interaction lookups.
- The frontend can bind a dedicated Vapi assistant ID via `NEXT_PUBLIC_VAPI_DRUG_CHECKER_ASSISTANT_ID`.
- If indexed payloads miss `drug_a` / `drug_b`, backend now infers names from transcript and enriches drug descriptions using openFDA labels API.

Environment flags in `backend/.env`:
- `QDRANT_DRUG_COLLECTION_NAME=medivoice_drugs`
- `DRUG_REFERENCE_API_ENABLED=true|false`
- `DRUG_REFERENCE_TIMEOUT_SECONDS=3.5`

## LLM + Qdrant Hybrid Triage
- Triage endpoints use Qdrant retrieval as grounding context.
- If LLM is enabled and API key is provided, final triage output is generated by LLM using transcript + Qdrant matches.
- If LLM is unavailable, backend falls back to deterministic triage logic.

Environment flags in `backend/.env`:
- `LLM_ENABLED=true|false`
- `GOOGLE_API_KEY=<your-gemini-api-key>`
- `GEMINI_API_KEY=<your-gemini-api-key>`
- `LLM_MODEL=gemini-2.5-flash`
- `OPENROUTER_API_KEY=<your-openrouter-api-key>`
- `OPENROUTER_MODEL=google/gemma-4-26b-a4b-it:free`

Report analysis prefers OpenRouter when `OPENROUTER_API_KEY` is set, and falls back to Gemini if OpenRouter is unavailable.

## Run locally
1. Create a virtual environment.
2. Install dependencies from `requirements.txt`.
3. Copy `.env.example` to `.env` and fill in credentials.
4. Start the app with Uvicorn or the helper script.

Example:

```bash
..\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

PowerShell helper:

```powershell
./run-backend.ps1
```

## Index symptom cases into Qdrant
To populate Qdrant with seed medical cases:

```powershell
Set-Location backend
..\.venv\Scripts\python.exe scripts\index_qdrant.py --recreate
```

Optional custom source file:

```powershell
..\.venv\Scripts\python.exe scripts\index_qdrant.py --source .\my_cases.json --recreate
```

## Index scheme coverage collection (PM-JAY / CGHS / ESIC)
To power post-triage scheme auto-surface cards, index scheme records into the dedicated Qdrant scheme collection:

```powershell
Set-Location backend
..\.venv\Scripts\python.exe scripts\index_qdrant_schemes.py --recreate
```

Optional custom source file:

```powershell
..\.venv\Scripts\python.exe scripts\index_qdrant_schemes.py --source .\my_schemes.json --recreate
```

## Index drug interaction collection
To improve drug checker accuracy and avoid unknown drug labels, index interactions with normalized `drug_a` and `drug_b` fields:

```powershell
Set-Location backend
..\.venv\Scripts\python.exe scripts\index_qdrant_drugs.py --recreate
```

Optional custom source file:

```powershell
..\.venv\Scripts\python.exe scripts\index_qdrant_drugs.py --source .\my_drug_interactions.json --recreate
```

Expected JSON format (`my_drug_interactions.json`):

```json
[
	{
		"drug_a": "Ibuprofen",
		"drug_b": "Warfarin",
		"interaction_severity": "Red",
		"safe_dosage": "Avoid co-use unless clinician-directed.",
		"recommendation": "High bleeding risk; consider alternatives.",
		"summary": "NSAID-anticoagulant combination can increase bleeding.",
		"warnings": ["Bleeding risk", "Watch for bruising"]
	}
]
```

Expected JSON format (`my_schemes.json`):

	{

## No hits troubleshooting
If the triage panel shows "No retrieval hits were returned for this triage turn", check the following in order.

1. Confirm Qdrant is running and reachable from backend:

```powershell
Invoke-RestMethod -Uri "http://localhost:6333/collections" -Method GET
```

2. Re-index seed cases into the active collection:

```powershell
Set-Location backend
..\.venv\Scripts\python.exe scripts\index_qdrant.py --recreate
```

3. Verify collection and vector size alignment in backend `.env`:
- `QDRANT_COLLECTION_NAME` must match the collection you indexed.
- `QDRANT_VECTOR_SIZE` must match the vectors in that collection.
- If vector size or embedding model changed, re-run indexing with `--recreate`.

4. Validate retrieval directly through API:

```powershell
Invoke-RestMethod -Uri "http://localhost:8000/knowledge/search" -Method POST -ContentType "application/json" -Body '{"transcript":"fever and sore throat","languageCode":"en-US","topK":3}'
```

5. If response has empty `matches`:
- Confirm backend uses the same Qdrant URL and API key as your running instance.
- Confirm indexed payloads include searchable symptom text (`text` or summary-derived text in the indexing script).
- Restart backend after `.env` changes.

Expected JSON format (`my_cases.json`):

```json
[
	{
		"problem_label": "Upper Respiratory Tract Infection",
		"severity_score": "Green",
		"red_flags": ["High persistent fever", "Breathing difficulty"],
		"specialty": "General Practitioner",
		"summary": "Mild sore throat and cough",
		"text": "I have sore throat and mild cough"
	}
]
```

## Frontend integration
The web app uses the Vapi web SDK pattern:

```ts
import Vapi from '@vapi-ai/web';

const vapi = new Vapi(process.env.NEXT_PUBLIC_VAPI_PUBLIC_KEY!);
await vapi.start(process.env.NEXT_PUBLIC_VAPI_ASSISTANT_ID!);
```

Set the frontend values in `.env.local`:

- `NEXT_PUBLIC_TRIAGE_API_BASE_URL`
- `NEXT_PUBLIC_VAPI_PUBLIC_KEY`
- `NEXT_PUBLIC_VAPI_ASSISTANT_ID`
- `NEXT_PUBLIC_VAPI_DRUG_CHECKER_ASSISTANT_ID`
